# 第一个菜单

这是最少出错的流程：创建元素、整理层级，然后在你堆出一坨诅咒盒子之前停手。

RZMenu 是 Blender 插件助手，不是独立导出器。它帮你构建菜单数据和 UI 逻辑，最终结果还是得走完后面的模组流程。

## 1. 打开编辑器

在 Blender 里打开 RZMenu 面板，然后启动编辑器。那个视口就是你放菜单元素的地方。

在空白处右键，使用 **Add**。

![Add menu](guide-media/editor-add-menu.png)

第一个元素通常是 **Anchor** 或 **Container**。它会成为其它所有东西的底座。

## 2. 认识基础元素类型

**Container** 是通用盒子。它负责分组，也可以承载自定义逻辑。

**Anchor** 和 Container 很像，但它更偏向用来移动和定位。它很适合作为屏幕或面板的根节点。

**Button** 是主要的可点击元素。它可以绑定变量、开关和其它逻辑。

**Slider** 会写入平滑数值，通常范围是 `0.0` 到 `1.0`。

**Text** 走的是另一条渲染路径。不要以为图片规则和文字规则会一样。

**Grid Container** 以前是懒人自动摆子节点的方法。重定位重构之后它坏了，维护者也没修。除非你喜欢死路，否则别碰它。

## 3. 整理树结构

Outliner 显示的是父子层级。把元素拖到父节点下面可以分组，拖出来就能让它们独立。

![Outliner pages](guide-media/outliner-pages.png)

多页菜单建议用独立根面板，再用页面变量控制显示。

## 4. 给页面用隔离根

如果一个面板就是一页，请在那个根元素上启用 page 或 isolation 选项。这样大型菜单会好管得多。

已知的毛病：如果页面第一次没渲染出来，切走再切回来。某些版本里刷新 bug 还在。作者还莫名其妙地很爱这个 bug，不太想把时间花在找隐藏代码路径上。

## 5. 收尾状态

到这里你应该已经有了：

- 一个根 Anchor 或 Container；
- 几个 Button、Text 块或 Slider；
- Outliner 里干净的层级；
- 一套驱动 UI 的变量计划。

下一步：绑定真正的开关和贴图槽。

如果你点了之后它有反应，哪怕只是看起来卡了一下，也算进展。站起来喊一句：`VICTORY`。
