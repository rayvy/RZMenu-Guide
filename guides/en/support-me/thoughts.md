# My Thoughts

Development of RZMenu has always been a balance between solving practical modding issues and fighting the limitations of third-party engine integrations.

Here are a few raw reflections on this project:

- **Blender as an IDE:** Building complex layouts inside Blender's viewport is challenging. The Python API is powerful but not designed for high-performance responsive UI layouts.
- **The Modding Community:** Modders are a unique audience. They range from pure beginners who don't know what a file extension is, to advanced technical artists who write custom shaders. Serving both is almost impossible.
- **Burnout:** When you spend weeks fixing a boundary offset bug only to realize a game update broke another unrelated pointer, you want to close the editor and never open it again.
- **Keep it Simple:** The best menus are those that rely on clean, simple logic. Overcomplicating variable bindings is the easiest way to end up with a layout that is impossible to debug.

Thank you for reading, and thanks to everyone who helps test and improve the tool.
